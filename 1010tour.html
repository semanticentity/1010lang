$1010

A memory-mapped audio primitive. 180-byte core. Infinite leverage.

Sound is memory. Memory is bytes. Bytes are yours.
POKE $1000, hear kick. That's it. That's the language.

Philosophy: Tiny core, explicit extensions. Like htmx—extend HTML, don't replace it. $1010 extends memory, doesn't hide it. A student's first program is 2 lines. ADSR, tuning, effects load as packs when you need them.

Core Concept

$1010 is a memory-mapped audio coprocessor. Write bytes to addresses, hear sound. No API, no drivers, no abstraction.

; Make a kick drum
POKE $1000, 1      ; gate ON at step 0
POKE $1500, 3      ; start sequencer
; That's it. Kick plays.

The entire spec fits in <300 BYTES of memory. A full song fits in a tweet.

Memory Map

Audio lives at $1000-$15FF. Everything is a byte. Core is ~180 bytes.

Tuning Modes

ModeDescriptionTeaching
12-TETEqual temperament, 2^(n/12)Default. Compromise tuning for key changes.
JUSTPure ratios (5:4 third, 3:2 fifth)Why thirds sound "off" in 12-TET. Beatless intervals.
PYTHPythagorean (perfect fifths stack)Why we need temperament. Wolf intervals.
COLUNDIAleksi Perälä's 47-frequency sequenceMusic beyond the grid. Non-octave tuning.

Primitives

Everything in $1010 reduces to these primitives:

BYTE8 bits. 0-255 unsigned, or -128 to +127 signed (two's complement).
ADDRESS16-bit pointer into memory. Written as $XXXX hex.
GATE0 = silent, >0 = trigger. For melodic voices, value = MIDI note.
STEPOne of 16 positions (0-F). Sequencer reads gates at each step.
TICKOne step advance. Rate = 60/BPM/4 seconds (16th notes).
VOICESound generator. 5 total: kick, snare, lead, bass, noise.
PATTERN16 bytes of gate data. One pattern per voice.

WASM Mapping

$1010 compiles to WebAssembly. Memory-mapped addresses become WASM linear memory.

;; WASM memory layout (64KB page)
(memory (export "memory") 1)

;; Address constants
(global $KICK_GATE i32 (i32.const 0x1000))
(global $SEQ_CTRL  i32 (i32.const 0x1500))
(global $SEQ_TEMPO i32 (i32.const 0x1501))

;; Read/write primitives
(func $read (param $addr i32) (result i32)
  (i32.load8_u (local.get $addr)))

(func $write (param $addr i32) (param $val i32)
  (i32.store8 (local.get $addr) (local.get $val)))

WASM Exports

ExportSignatureFunction

Syscalls / Imports

WASM imports audio triggers from the host. The host handles synthesis.

;; Host must provide these
(import "audio" "triggerKick"  (func (param i32)))           ; pitch
(import "audio" "triggerSnare" (func (param i32 i32)))       ; tone, snap
(import "audio" "triggerLead"  (func (param i32)))           ; note
(import "audio" "triggerBass"  (func (param i32 i32)))       ; note, fm
(import "audio" "triggerNoise" (func (param i32)))           ; decay

Tick Cycle

// Host calls tick() at BPM rate
setInterval(() => {
  if (wasm.tick()) {           // returns 1 if playing
    updateUI(wasm.getStep());
  }
}, (60 / bpm / 4) * 1000);     // 16th note interval

Division of labor:

  • WASM: sequencer state, memory, step logic
  • Host: timing, audio synthesis, UI

Compile Targets

The $1010 DSL compiles to multiple backends:

MTMC-16 Assembly

Native target. Runs on Carson Gross's teaching VM. Uses sys memcopy for pattern loading.

WebAssembly (.wat)

Browser target. Linear memory maps 1:1. Host provides Web Audio synthesis.

C

Embedded target. uint8_t arrays, function calls. Portable to Arduino, Pi Pico, etc.

Rust

Safe embedded. Same memory model, stricter types. no_std compatible.

Intel HEX

Raw binary. Flash directly to EEPROM. Old school ROM distribution.

Same 260-byte song, five different machines.

Extending $1010

The spec is minimal by design. Extension points:

Add a Voice

; Reserve $1400-$14FF for new voice
$1400: gate[16]    ; trigger pattern
$1410: param1[16]  ; voice-specific
$1420: param2[16]

; Add import to WASM
(import "audio" "triggerMyVoice" (func (param i32 i32)))

Add Effects

; Effects region at $1600
$1600: FX_REVERB   ; 0-127
$1601: FX_DELAY    ; 0-127
$1602: FX_FILTER   ; 0-255 (cutoff)

Longer Patterns

; 32-step mode: use $1504 as length register
$1504: SEQ_LENGTH  ; 16, 32, or 64

; Second page of patterns at $1800

Embed in Your App

// Minimal embedding
const mem = new Uint8Array(65536);
const KICK = 0x1000;

mem[KICK + 0] = 1;  // step 0: kick
mem[KICK + 4] = 1;  // step 4: kick

// Your tick loop
function tick(step) {
  if (mem[KICK + step]) playKick();
  if (mem[SNARE + step]) playSnare();
  // ...
}

Example: XEnv Pack (ADSR)

; Lead voice: slow attack pad sound
POKE $1068, 200        ; A = slow attack (~800ms)
POKE $1069, 50         ; D = short decay
POKE $106A, 180        ; S = high sustain (70%)
POKE $106B, 150        ; R = medium release (~600ms)

; Bass voice: punchy pluck
POKE $106C, 5          ; A = instant attack
POKE $106D, 80         ; D = medium decay
POKE $106E, 50         ; S = low sustain (20%)
POKE $106F, 40         ; R = quick release

; Each value 0-255, scaled by synth (typically /255 * max_time)

Example: XTuning Pack

; Switch to Just Intonation
POKE $10A0, 1          ; mode = JUST

; Or try Colundi (47 non-12TET frequencies)
POKE $10A0, 3          ; mode = COLUNDI

; 432 Hz tuning (A4 = 432 instead of 440)
POKE $10A1, $B0        ; low byte of 432
POKE $10A2, $01        ; high byte

; Detune lead +7 cents, bass -7 cents (instant thickness)
POKE $10A5, 135        ; lead: 128+7 = 135
POKE $10A6, 121        ; bass: 128-7 = 121

Example: XTempo Pack

; 16-bit BPM for extreme tempos
$1700: TEMPO_LO        ; Low byte
$1701: TEMPO_HI        ; High byte

; 300 BPM = $2C + ($01 * 256)
POKE $1700, $2C        ; 44
POKE $1701, $01        ; 256
; Total: 44 + 256 = 300 BPM

Example: XEuclidean Pack

; E(k,n) = distribute k hits across n steps
; Famous rhythms from GCD math:

E(3,8)  = x..x..x.    ; Cuban tresillo
E(5,8)  = x.xx.xx.    ; Cinquillo
E(5,16) = x..x..x..x..x...  ; Bossa nova
E(7,16) = x.x.x.xx.x.x.x..  ; West African

; Set kick to E(5,16):
POKE $1B00, 5          ; 5 hits
POKE $1B01, 16         ; 16 steps
POKE $1B0F, 1          ; enable

Example: XFollow Pack (from Ableton Live)

; After 4 plays, 70% chance jump to random slot
POKE $1C01, 4          ; play 4 times
POKE $1C02, 3          ; action = random
POKE $1C03, 70         ; 70% chance
POKE $1C00, 1          ; enable for kick

Example: XPatch Pack (from Max/MSP)

; Route LFO output to filter cutoff
POKE $1D10, $46        ; source = LFO1_VALUE ($1B46)
POKE $1D11, $1B
POKE $1D12, $21        ; dest = FX_FILTER_CUTOFF ($1821)
POKE $1D13, $18
POKE $1D14, 64         ; unity gain
POKE $1D00, 1          ; 1 patch active
